# 캡스톤디자인 결과보고서

## 1. 캡스톤디자인 개요

### 1.1 캡스톤디자인 배경

현대 취업 시장에서 포트폴리오는 구직자의 핵심 마케팅 도구로 자리잡고 있습니다. 하지만 채용공고마다 요구하는 기술 스택과 역량이 다르기 때문에, 매번 포트폴리오를 수정하고 재정렬하는 것은 시간과 노력이 많이 소요되는 문제가 있습니다.

특히 IT 분야에서는 기술 스택이 빠르게 변화하고, 각 기업마다 선호하는 기술과 도구가 다르기 때문에, 하나의 고정된 포트폴리오로는 다양한 채용공고에 효과적으로 대응하기 어렵습니다. 이러한 문제를 해결하기 위해 AI 기반의 지능형 포트폴리오 재정렬 시스템을 개발하게 되었습니다.

### 1.2 캡스톤디자인 목표

- **주요 목표**: 채용공고 분석을 통한 자동 포트폴리오 재정렬 시스템 개발
- **기술적 목표**:
  - 채용공고에서 자격요건/우대사항 자동 추출
  - 키워드 기반 스킬/프로젝트 매칭 알고리즘 구현
  - 가중치 기반 정렬 시스템 개발
- **사용자 경험 목표**:
  - 원클릭 포트폴리오 재정렬 기능
  - 직관적인 웹 인터페이스 제공
  - 실시간 미리보기 기능

### 1.3 캡스톤디자인 관련 조사(연구)

기존 포트폴리오 관리 도구들을 조사한 결과, 대부분 정적 포트폴리오 생성에 집중되어 있었습니다. 주요 차별점은 다음과 같습니다:

- **기존 도구**: 템플릿 기반 정적 포트폴리오 생성
- **본 프로젝트**: 채용공고 분석 기반 동적 재정렬 시스템
- **특이성**: AI 키워드 추출과 규칙 기반 폴백 시스템의 하이브리드 접근
- **대표성**: 취업 준비생의 실제 니즈를 반영한 실용적 솔루션

#### 4.2.3 데이터베이스 설계서

**ERD (Entity Relationship Diagram)**:

```
┌─────────────┐    ┌─────────────────┐
│    Users    │    │   Portfolios    │
│             │    │                 │
│ - id (PK)   │◄──►│ - id (PK)       │
│ - email     │    │ - owner_email   │
│ - password  │    │ - kind          │
│ - created_at│    │   (BASIC/REMODEL)│
└─────────────┘    │ - data_json     │
                   │ - updated_at    │
                   └─────────────────┘
```

**테이블 상세 설계**:

**Users 테이블**:

```sql
create table public.users (
  id bigint generated by default as identity not null,
  email character varying(255) not null,
  name character varying(255) not null,
  password_hash character varying(255) not null,
  constraint users_pkey primary key (id),
  constraint idx_users_email unique (email),
  constraint uk_users_email unique (email)
) TABLESPACE pg_default;
```

**Portfolios 테이블**:

```sql
create table public.portfolio (
  id bigint generated by default as identity not null,
  data_json text not null,
  kind character varying(16) not null,
  owner_email character varying(190) not null,
  updated_at timestamp with time zone not null,
  constraint portfolio_pkey primary key (id),
  constraint portfolio_kind_check check (
    (
      (kind)::text = any (
        array[
          ('BASIC'::character varying)::text,
          ('REMODEL'::character varying)::text
        ]
      )
    )
  )
) TABLESPACE pg_default;
```

**Supabase 특성**:

- **클라우드 기반**: PostgreSQL을 클라우드에서 관리
- **자동 백업**: 데이터 자동 백업 및 복구
- **실시간 기능**: Supabase Realtime을 통한 실시간 업데이트 가능
- **REST API**: 자동 생성되는 REST API 제공
- **인증 시스템**: Supabase Auth와 연동 가능

**데이터 저장 방식**:

- **기본 포트폴리오**: `kind = 'BASIC'`으로 저장
- **재정렬된 포트폴리오**: `kind = 'REMODEL'`로 저장
- **JSON 데이터**: `data_json` 필드에 전체 포트폴리오 데이터를 JSON 형태로 저장
- **관계**: `owner_email`을 통해 사용자와 연결

#### 4.2.4-1 프로젝트 설명 추천 기능 (규칙 기반 폴백)

본 섹션은 GPT 호출 실패 시 자동으로 사용되는 규칙 기반 폴백 추천을 설명합니다. API 오류/타임아웃/비정형 응답 등 예외 상황에서 아래 절차로 프로젝트 설명 추천을 수행합니다.

- 입력: 채용공고 텍스트(자격요건/우대사항), 포트폴리오의 스킬 목록과 각 프로젝트의 기술 스택/설명
- 처리 절차:

  1. 규칙 기반 키워드 추출: 기술/도메인/방법론 패턴과 토큰화를 활용하여 키워드와 가중치를 산출
  2. 스킬 매칭 스코어 계산: 키워드-스킬 토큰 일치도 기반 가중 합산 점수 산출 및 정렬
  3. 프로젝트 매칭 스코어 계산: 프로젝트 기술 스택과 설명 문장에 대한 키워드 매칭(기술/텍스트 분리) 점수 산출 및 정렬
  4. 설명 추천 생성: 매칭된 키워드와 가중치에 따라 다음 형태의 보강안을 제시
     - 하이라이트: 설명 내 일치 키워드 강조(예: FastAPI, RAG, Kubernetes 등)
     - 근거 기반 보강 문장 템플릿 추천: “키워드 기반 성과/지표/역할” 형태의 예시 문장 목록
       - 예) "RAG 기반 검색 파이프라인을 설계/구현하여 질의 응답 정확도 X% 향상"
       - 예) "Kubernetes 환경에서 End-to-End ML 파이프라인 자동화로 배포 시간 Y% 단축"
     - 불일치 시 권고: 공고의 핵심 키워드 중 누락된 항목(예: MLOps, Observability 등)에 대한 보완 제안

- 출력: 정렬된 프로젝트 목록과 각 프로젝트에 대한 추천 포인트 리스트(하이라이트 및 보강 문장 예시)

진행 중 로깅: 매칭된 키워드/가중치, 스킬/프로젝트 별 총점이 로그로 출력되어 추천의 근거를 추적할 수 있습니다. AI를 사용하지 않을 때는 AI 관련 실패 로그가 출력되지 않도록 비활성화 설정과 조건 분기 처리가 되어 있습니다.

#### 4.2.4-2 프로젝트 설명 추천 기능 (OpenAI GPT 기반, 사용자 트리거)

본 기능은 포트폴리오 폼 페이지의 "GPT 설명 추천" 버튼 클릭 시 항상 GPT API를 호출합니다(사용자 트리거). 규칙 기반으로 추출·정렬된 근거를 LLM에 컨텍스트로 제공하여 자연스러운 한국어 프로젝트 설명 보강안과 대체 설명을 생성합니다. LLM 응답 품질이 낮거나 실패 시 자동으로 규칙 기반 출력으로 폴백합니다.

- 입력 컨텍스트(예):

  - 채용공고에서 추출된 키워드(+가중치, 종류)
  - 현재 프로젝트의 기술 스택 및 기존 설명 문장
  - 매칭 결과(기술 매칭/텍스트 매칭 점수, 하이라이트 후보)

- 프롬프트 전략(요지):

  - 역할 지시: “채용공고와 포트폴리오의 적합도를 최대화하도록 프로젝트 설명을 한국어로 간결하게 보강하라.”
  - 제약: “허위 성과 금지, 실제 기술/역할만 반영, 2~4문장, 불필요한 수식어 최소화, 핵심 키워드 포함.”
  - 포맷: JSON 객체로 반환하도록 지정하여 파싱 안정성 확보

- 요청 예시(JSON):

```json
{
  "model": "gpt-4o-mini",
  "input": [
```

#### 4.2.9 알고리즘 설계

**키워드 추출 알고리즘**:

```java
public List<Keyword> extractKeywords(String text) {
    List<Keyword> keywords = new ArrayList<>();

    // 1. 패턴 기반 키워드 추출
    for (String pattern : TECH_PATTERNS) {
        if (text.contains(pattern)) {
            keywords.add(new Keyword(pattern, calculateWeight(pattern), TECH));
        }
    }

    // 2. 단어 기반 키워드 추출
    String[] words = text.split("\\s+");
    for (String word : words) {
        if (isTechRelated(word)) {
            keywords.add(new Keyword(word, 0.5, TECH));
        }
    }

    // 3. 중복 제거 및 가중치 통합
    return deduplicateKeywords(keywords);
}
```

**재정렬 알고리즘**:

```java
public List<String> reorderSkills(List<String> skills, List<Keyword> keywords) {
    Map<String, Double> scores = new HashMap<>();

    // 1. 각 스킬의 매칭 점수 계산
    for (String skill : skills) {
        double score = 0.0;
        for (Keyword keyword : keywords) {
            if (containsToken(skill, keyword.getTerm())) {
                score += keyword.getWeight();
            }
        }
        scores.put(skill, score);
    }

    // 2. 점수 기준으로 정렬
    return skills.stream()
        .sorted((s1, s2) -> Double.compare(scores.get(s2), scores.get(s1)))
        .collect(Collectors.toList());
}
```
